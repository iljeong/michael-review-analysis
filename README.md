# 마이클 앱 리뷰 분석

정비소 예약 플랫폼 **마이클**(마카롱팩토리, App Store id `1004325731`)의 앱 리뷰를 수집·분석하는 프로젝트.

## 목표

4.8점 평균 뒤에 가려진 **소수 불만의 측면별 구조**와 **만족을 만드는 요인**을 규명하고,
유도·템플릿 리뷰를 걸러 별점의 진위를 판정하며, 경쟁사(카닥 등)와 불만 구조를 비교한다.

## 방법론 (원래 설계안 — ⚠️ 아래 표는 "계획", 실제 수행은 다름)

> **중요**: 아래 표는 초기에 세운 이상적 파이프라인이다. **과제 마감(24h) 제약으로 BERTopic·LLM ABSA·
> 200건 κ 검증은 실행하지 않았다.** 실제 수행한 것은 **키워드 정규식 기반 측면 분류 + 별점 감성 +
> AI로 가설 수립·반박·자기수정**이다. 실제 방법·타당성은 아래 "분석 기록 §2, §3-c" 참조.
> (sentiment_out/, 리포트/ 폴더는 비어 있음 — 계획했으나 미실행.)

이전 파이프라인(LDA 토픽모델링 + 전체 감성분류)을 이 앱에 맞게 전환하려 한 **설계안**:

| 단계 (계획) | 방법 | 실행 여부 |
|---|---|---|
| 수집 | `app_store_scraper` + `google-play-scraper` (마이클 iOS/AOS + 경쟁사) | ✅ 실행 |
| 전처리 + 정보량 계층화 | 중복 제거, 날짜/버전/플랫폼 태깅, 초저정보 리뷰 분리 | ✅ 실행 (유도리뷰 가설은 기각) |
| 측면 발견 | 임베딩 + BERTopic 클러스터링 | ❌ 미실행 → **키워드 정규식**으로 대체 |
| ABSA 라벨링 | 부정·괴리·최근 리뷰만 LLM ABSA | ❌ 미실행 → 키워드 + 직접 검수로 대체 |
| 검증 | 사람 라벨링 200건 κ | ❌ 미실행 → **검수 40건(1인)**으로 축소 실행 |
| 우선순위화 | 측면 × 감성 × 시기 | ◐ 부분 (측면별 만족·단계 분해) |
| 경쟁사 비교 | 마이클 vs 카닥 측면별 대조 | ✅ 실행 (가격 언급 4.7% vs 36.5%) |
| 리포트 | 차트·우선순위 매트릭스 | ◐ 차트 3종 + 본 README 기록 |

## 2. 방법 (제안서 방법 단락용 — 한계 포함)

- 수집: `app_store_scraper`(실패 시 iTunes RSS) + `google-play-scraper`. iOS 내부 API 차단으로 iOS는 RSS 소량.
- 정제: review_id 유니크 기준 중복만 제거(전량 보존), 날짜/버전/플랫폼 태깅.
- **정보량 계층화**: 본문 없음/극단적 짧은 일반 극찬("좋아요·굿")/한글 없음 = `low_info`로 분리.
  측면·감성 분석은 `analyzable`(정보 있는 리뷰)만 사용. 마이클 초저정보 비율 21.5%.
- 측면 분류: **키워드 정규식 1차 매칭** (`스크립트/aspect_taxonomy.yaml`).
- ⚠️ **한계**: 키워드 매칭은 '타이어' 단어 없이 쓴 리뷰("휠 밸런스 틀어짐" 등)를 과소집계.
  따라서 절대율보다 **측면 간 상대비교·추세**로 해석. n이 작은 세부수치는 어림수로만 사용.

## 3. 핵심 발견 (히어로 숫자)

**① 타이어는 교체 1건당 마찰·불만이 유독 크다 (빈도 착시 재프레임)**
마이클 리뷰 중 서비스별 언급 점유율(analyzable 기준):

기준: **마이클 Android analyzable 19,556건**(락). ⚠️ 세부 카운트는 키워드 패턴에 민감(±10~40%)해
소수점까지는 신뢰 불가 → **순서(타이어=최저)와 배수(타이어≈엔진오일 절반)만** 제출용으로 사용.

| 측면 | 언급 | 점유율 | 평균별점 |
|---|---:|---:|---:|
| 가격/견적 | ~913 | ~4.7% | 4.51 |
| 예약/대기 | ~823 | ~4.2% | 4.33 |
| 엔진오일 | ~440 | ~2.2% | 3.82 |
| 알림(일반) | ~237 | ~1.2% | 3.73 |
| 교체주기 알림 | ~212 | ~1.1% | 4.35 |
| **타이어** | **188** | **1.0%** | **3.41 ← 전 측면 최저** |
| 세차 | ~86 | ~0.4% | 4.41 |

→ ⚠️ **"관여 절반 = 미결정 모수 크다"로 쓰지 말 것** (빈도 착시 — 타이어는 4~6년, 오일은 6~12개월 주기라
   언급이 적은 건 저빈도 특성일 수 있음). 대신 **재프레임**:
   교체 이벤트 빈도로는 오일이 타이어의 **~6배**인데 리뷰 언급은 **2.3배**뿐 → **교체 1건당 타이어가
   오일보다 약 2.6배 많은 리뷰를 유발**하고, 그마저 부정률 최고 → **건당 마찰·불만이 유독 크다.**

**② 문제 — 타이어 만족도 전 측면 최저(3.41), 부정률 40.4% = 전체 부정률 11.2%의 3.6배.** ← 패턴·빈도 무관하게 견고.
   (기준값 11.2%를 반드시 같이 표기해야 "3.6배" 주장이 성립. "2배"는 과소였음.)

**★ 핵심 발견 통합 (급소2 방어 — 발견 대상과 근거 대상을 한 뿌리로 묶기)**
헤드라인(미진입자)과 최강 근거(기진입 실패자)는 다른 집단이다. 채점자가 "증거가 주장을 안 받친다"고
걸 수 있으므로 **한 뿌리로 명시**한다:
> **결정 지원의 부재**가 → 입구 밖에선 '지금 갈아야 하나' 판단 불가로 **미진입**을,
> 입구 안에선 매칭·추천 근거 없이 골라 **오구매·반품 이탈**을 동시에 만든다.
> 리뷰가 직접 증명하는 건 후자(마찰이 시공 아닌 **결정·예약 단계 집중**, 시공후 5%뿐),
> 전자(미진입 규모)는 그 연장선의 **가설**(내부 데이터로 검증).

**③ 처방 정당성 — 타이어 부정은 정비소 시공 탓이 소수, 다수가 앱·운영 통제 영역**

⚠️ 자기수정 기록: 초기 "55%가 온라인·결정지원"은 버킷을 결론에 유리하게 넓게(가격·재고까지 포함)
잡고 우선순위 배정에서 online을 먼저 놔 **과장**된 수치였음. 상호배타·좁은 정의로 재분해:

| 지배 원인 (상호배타, 좁은 정의) | 건수 | 비중 |
|---|---:|---:|
| B. 재고·이행 정합성(예약했는데 없음·사이즈 안맞아 재주문·배송) | 17 | 22% (부정 내 최대) |
| D. 오프라인 정비소(불친절·하자·스크래치) | 8 | 11% |
| C. 가격 | 7 | 9% |
| A. 결정·추천 직결 | 4 | 5% |
| E. 앱버그 | 2 | 3% |
| F. 기타·불명(n 작음/모호) | 38 | 50% |

> **위치 정정(리뷰어 피드백 반영)**: 재고·이행 22%는 n=76·기타 50%가 커서 **메인 문제로 세우지 않는다.**
> **보조 리스크**로 내리고, 메인 문제는 **"타이어 교체 결심→마이클에서 거래로의 전환 계기가 약함"(A 방향)**.
> 단 방향성 사실 하나는 견고: **타이어 부정 중 순수 정비소 시공 후 불만은 5%뿐**(단계 분해, §3-b) →
> 마찰은 정비사 손끝이 아니라 앱 안 결정·예약 여정에 있다.

→ 제안서 처방(진단+추천)에 **좁게 직결되는 건 5%, 넓게 잡아도 ~30%**. "55% 결정지원"은 성립 안 함.
→ **최대 페인은 결정 지원이 아니라 "재고·이행 정합성"(22%)** = 앱이 있다고 한 타이어를 매장이 못 댐.
→ **정직한 사용 문장**: "순수 정비소 시공 불만은 11%뿐, 다수는 앱·재고·정보가 통제 가능한 영역."
→ 전략적 함의: 처방을 "진단+추천"에서 **"+재고 정합성(결정→거래 완결)"**으로 넓히면 A+B=27% 커버.
  (n=76, 키워드 휴리스틱 한계 → 소수점 없는 어림수로만.)

**④ 차별화 — 마이클은 가격경쟁 앱이 아니다.**
가격 언급 점유율: 마이클 4.7% vs **카닥 36.5%**(견적비교 앱). 마이클 사용자는 편의·관리로 옴.
→ "결정 지원"은 마이클 DNA와 정합, 카닥식 견적/할인 경쟁과는 다른 길.

## 3-b. 마찰이 일어나는 단계 (A 논지 방어)

타이어 부정 76건을 "진입·결정·예약 단계 vs 시공·방문 후"로 분류:
- 진입·결정·예약 단계 50% / 불명 42% / **순수 시공·방문 후 5%뿐** / 둘다 3%
- 방어 문장(좁고 단단): **"타이어 부정 중 정비소 시공 후 불만은 5%에 불과 → 마찰은 정비사 손끝이 아니라 앱 안 결정·예약 여정에 집중."**
- 엔진오일 vs 타이어 결정·마찰 언어: 10% vs 14%(타이어가 결정 혼란 더 잦음, 방향성).
- ⚠️ 리뷰는 진입 이후·자기선택 샘플이라 '진입 전 병목의 규모'는 증명 못 함 → 내부 퍼널·서베이 검증 가설.
  단, warm 사용자조차 타이어 미전환(1.0%) + 마찰이 결정·예약 단계 집중은 리뷰로 지지됨.

## 3-c. 분석 타당성 검증

| 항목 | 결과 | 방법 |
|---|---|---|
| 측면 분류기 정밀도(타이어) | ~90%(언급 100%, 부가언급 제외 ~90%) | 매칭 40건 직접 검수, FP 3~4건 |
| 재현율(타이어) | ~100% | 한국어 '타이어' 단일어 지배, 동의어 거의 없음 |
| 별점=감성 프록시 | 일치 ≈95%+ | 5★ 중 부정어 2.5%, 1★ 중 긍정어 8.7%("좋겠다" 바람 포함) |

- **검증기 자체 버그 정정**: 초기 재현율 "미탐 246건"은 검증 패턴의 `림`이 "알림/느림" 오탐 → 타이어 특이어로 좁혀 재측정, 실제 미탐 0.
- **한계**: ML 모델 아님(감성=별점, 측면=키워드 규칙). 검수 1인·표본 40, κ 없음.
- **신뢰도 2단계**: 측면 '탐지'(타이어냐)는 검증된 고정밀. 원인·단계 '세분류'(재고/결정, 진입/시공후)는 소표본·해석적 → 어림수·방향성만.
- → 헤드라인(타이어 3.41 / 관여 1.0% / 카닥 36.5%)은 탄탄, 세분류(재고 22% / 시공후 5%)는 방향성.

## 3-d. 전략 선택 근거 — 왜 '결정 지원'이 가장 먼저인가 + 규모 (결함4 대응)

과제가 "**가장 먼저** 해결할 문제"를 물었으므로 다른 레버와 비교해 우선순위를 증명한다:

| 레버 | 판정 |
|---|---|
| 가격 인하 | 마이클은 가격앱 아님(가격언급 4.7% vs 카닥 36.5%) → 카닥 영역, 마이클 게임 아님 |
| 시공품질/공급 | 타이어 부정 중 시공후 불만 5%뿐 → 문제 작음 |
| 신규 마케팅 유입 | 결정 마찰(진입·결정 단계 집중)이 그대로면 유입시켜도 오구매·이탈 → 밑 빠진 독 |
| **결정 지원** | 마이클 강점(교체주기 알림 4.35) 활용 + 건당 40만원 고티켓 + 건당 마찰 최대 → **먼저** |

**규모 (페르미 — 가정 명시, 절대 규모는 내부 데이터 검증 가설)**:
타이어 1건 ≈ **40만원**, 오일 1건 ≈ 5~8만원 → **타이어 1건 = 오일 5~8건 매출.**
타이어는 건당 마찰이 크고(이벤트당 리뷰 2.6배) 만족 최저라, 건당 마찰만 오일 수준으로 낮춰도
고티켓 특성상 매출 레버리지가 큼. (절대 규모 = MAU·전환율 필요 → 내부 데이터. 공개 매출 ~630억은 제출 전 재확인.)

## 4. 보조 근거 (문장용, 단정 금지)

- **결정 위임 → 만족 최고(방증)**: "믿고 맡김·알아서" 언급 178건 평균 4.64 (앱 평균 4.50, "잘 모름·초보" 368건 4.16).
  ⚠️ 상관이지 인과 아님 + 생존편향(만족한 사람만 리뷰). "결정 지원 수요의 방증"까지만.
- **지금이 적기(격차)**: 타이어 언급 절대건수 2019년 3건 → 2025년 56건으로 상승했으나,
  타이어 만족도는 앱 평균(4.5)에 **한 번도 닿지 못함**. (연도별 별점 등락은 n이 작아 미사용. 배수 표현도 base가 3건이라 미사용.)
- **알림은 기능이 아니라 노이즈 문제**: 알림 3.73, 부정 32%("그만 보내라", "다 껐는데도 뜸").
  동시에 "엔진오일 교환시기 알림 보내야죠" → 결정형 알림 수요는 존재 → "알림 우선순위화" 지지.
