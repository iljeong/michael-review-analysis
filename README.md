# 마이클 타이어 교체 리뷰 분석

마카롱팩토리 과제 전형의 **"마이클 타이어 교체 서비스 거래 확대 전략"**을 준비하기 위해 앱 리뷰를 수집하고 분석한 작업 기록입니다.

핵심 목적은 하나입니다.

> 타이어 거래 확대를 위해 가장 먼저 해결해야 할 문제가 무엇인지, 리뷰 데이터와 실제 사용 경험을 함께 놓고 판단한다.

---

## 1. 현재 결론

현재까지의 분석으로는 **가격 경쟁**보다 **타이어 교체 결심을 마이클 안에서 거래로 전환시키는 문제**가 더 중요해 보입니다.

마이클은 카닥처럼 견적 비교를 전면에 둔 앱이 아닙니다. 리뷰에서도 마이클 사용자는 가격보다 차량관리, 편의, 예약 경험을 더 많이 말합니다. 따라서 타이어 전략도 최저가 경쟁보다 **차량관리 맥락에서 타이어 교체 필요성을 인식시키고, 사용자가 마이클에서 바꾸겠다고 결심하게 만드는 방향**이 더 자연스럽습니다.

다만 리뷰만으로는 "진입 전 미결정 모수의 크기"를 직접 증명할 수 없습니다. 리뷰가 강하게 보여주는 것은 타이어에 진입한 사용자조차 만족도가 낮고, 불만이 결정·예약·정보 영역에 몰린다는 점입니다. 그래서 최종 제안서에서는 아래처럼 쓰는 것이 안전합니다.

> 리뷰 데이터는 타이어 거래의 진입 전 모수까지 직접 보여주지는 못한다. 대신 마이클 안에 들어온 타이어 사용자도 만족도가 낮고, 마찰이 결정·예약 단계에 집중된다는 점을 보여준다. 따라서 전략은 가격 할인보다 "타이어 교체 필요 판단 → 맞춤 추천 → 예약"으로 이어지는 결정 전환 퍼널을 강화하는 쪽이 타당하다.

## 2. 데이터 개요

| 소스 | 수집 리뷰 | 분석 가능 리뷰 | 평균 별점 |
|---|---:|---:|---:|
| 마이클 Android | 24,903 | 19,556 | 4.57 |
| 마이클 iOS | 50 | 50 | 3.36 |
| 카닥 Android | 7,266 | 6,050 | 4.40 |
| 카닥 iOS | 150 | 137 | 4.01 |
| **합계** | **32,369** | **25,793** | |

- 분석 backbone은 **마이클 Android 분석 가능 리뷰 19,556건**입니다.
- 기간은 2013년부터 2026년 7월까지입니다.
- iOS는 내부 API 수집이 막혀 RSS로만 일부 수집했습니다. 표본이 작고 최근 부정 리뷰가 섞일 가능성이 커서 별점 판단에는 보조로만 씁니다.
- 별점 평균은 수집된 공개 리뷰 기준입니다. 스토어에 표시되는 현재 평점과는 계산 방식이 다릅니다.

---

## 3. 분석 방법

실제로 수행한 방법은 아래와 같습니다.

1. `app_store_scraper`, `google-play-scraper`, iTunes RSS로 마이클·카닥 리뷰 수집
2. `review_id` 기준 중복 제거
3. 본문 없음, 극단적으로 짧은 일반 칭찬, 한글 없음 리뷰를 `low_info`로 분리
4. 분석 가능한 리뷰만 대상으로 키워드 정규식 기반 측면 분류
5. 별점 `<=3`을 부정 리뷰의 프록시로 사용
6. 핵심 가설을 AI로 세우고, CSV 재계산과 직접 검수로 반박·수정

초기에는 BERTopic, LLM ABSA, 사람 라벨링 200건 검증까지 계획했지만, 과제 마감 제약상 실행하지 않았습니다. 따라서 최종 제안서에는 **키워드 기반 리뷰 분석 + AI를 활용한 가설 검증·자기수정 과정**으로 설명하는 것이 맞습니다.

## 4. 핵심 숫자

기준: 마이클 Android 분석 가능 리뷰 19,556건

| 측면 | 언급 수 | 점유율 | 평균 별점 | 부정률 |
|---|---:|---:|---:|---:|
| 차량관리 | 6,756 | 34.5% | 4.52 | 10.4% |
| 편의/간편 | 2,437 | 12.5% | 4.78 | 3.7% |
| 가격/견적 | 936 | 4.8% | 4.50 | 12.0% |
| 예약/대기 | 844 | 4.3% | 4.32 | 17.7% |
| 엔진오일 | 419 | 2.1% | 3.80 | 32.2% |
| **타이어** | **188** | **1.0%** | **3.41** | **40.4%** |
| 알림/팝업 | 223 | 1.1% | 3.79 | 30.9% |
| 세차 | 46 | 0.2% | 4.11 | 17.4% |

읽는 법:

- 타이어 언급은 1.0%로 작지만, 평균 별점은 3.41로 가장 낮습니다.
- 타이어 부정률은 40.4%입니다. 전체 앱 평균보다 훨씬 높고, 엔진오일보다도 높습니다.
- 타이어 언급 수가 작다고 해서 중요도가 낮다고 단정하면 안 됩니다. 타이어는 교체 주기가 길고, 한 번의 거래 금액이 큰 서비스입니다.

---

## 5. 카닥 비교

| 앱 | 가격/견적 언급 | 점유율 |
|---|---:|---:|
| 마이클 Android | 936 | 4.8% |
| 카닥 Android | 2,231 | 36.9% |

카닥은 가격·견적 비교 맥락이 강합니다. 반면 마이클은 차량관리와 예약 편의 맥락이 더 강합니다.

따라서 마이클 타이어 전략을 카닥식 가격 비교나 할인 경쟁으로 잡으면 포지션이 흐려질 수 있습니다. 마이클의 강점은 사용자가 이미 등록한 차량 정보, 정비 이력, 교체주기 관리입니다. 타이어도 이 맥락에서 풀어야 합니다.

---

## 6. 타이어 불만에서 보이는 패턴

타이어 리뷰 188건 중 부정 리뷰는 76건입니다. 원문을 보면 반복되는 불만은 다음과 같습니다.

- 내 차에 맞는 타이어인지 확신하기 어렵다.
- 선택 가능한 타이어 종류가 적거나 추천 이유가 불충분하다.
- 할인·쿠폰·추가비·반품비에 대한 신뢰가 낮다.
- 예약 후 재고, 배송, 장착 가능 여부에서 불일치가 생긴다.
- 일부는 정비소 응대나 현장 경험으로 이어진다.

여기서 주의할 점이 있습니다. 초기에 "부정 리뷰의 55%가 온라인 결정지원 문제"라고 해석했지만, 재분류해보니 과장이었습니다. 좁게 보면 결정·추천 직결 이슈는 작고, 재고·이행 정합성이나 가격·반품비, 기타 불명 이슈가 섞여 있습니다.

따라서 최종 제안서에서는 세부 비중을 강하게 주장하지 않는 것이 좋습니다. 대신 다음 문장이 더 안전합니다.

> 타이어 부정은 순수 시공 품질 하나로 설명되지 않는다. 사용자가 고르고, 예약하고, 장착을 확정하기 전후의 정보·추천·가격·이행 신뢰가 함께 흔들린다.

---

## 7. 한계

- 리뷰는 이미 앱에 진입한 사용자의 자기선택 표본입니다. 진입 전 미결정 사용자의 규모는 내부 퍼널 데이터나 설문이 있어야 확인할 수 있습니다.
- 측면 분류는 ML 모델이 아니라 키워드 정규식입니다. 절대 수치보다 방향성과 비교에 더 적합합니다.
- iOS 표본은 작고 최근 리뷰 중심입니다. 핵심 판단은 Android 리뷰를 기준으로 합니다.
- 별점을 감성 프록시로 썼기 때문에 리뷰 문맥과 완전히 일치하지 않을 수 있습니다.
